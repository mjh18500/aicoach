Rate Limits

Gmail API: 250 quota units/second. Add a "Wait" node (1-2 sec) between batch email calls. Claude Haiku: very generous, rarely hit limits.

Area 02

UI/UX Design

Interface patterns, a live chat mockup, and design decisions that make your prototype feel production-ready.

Design Principles

1

Calm by Default

Your companion touches email, mood, and jobs — all high-anxiety areas. The UI must feel grounding: dark or neutral tones, generous whitespace, no red badges or notification spam.

2

One Thing at a Time

Never show more than one primary action. The morning brief is one card. The mood check-in is one question. Cognitive load is the enemy.

3

Personality Visible

Show the AI's name and avatar consistently. Users bond with named characters. The tone of messages should feel like one consistent person, not a chatbot.

4

Progressive Disclosure

Start with a summary card. Let users tap to expand. Never dump a wall of text. The companion should surface what matters and hide the rest.

4 Interface Options

Telegram Bot

Fastest

Best for prototype. No UI to build. Use bot commands (/brief, /mood, /jobs). Users already have Telegram. Works on all devices instantly.

Web Chat (React)

Best Demo

Deployed on Vercel in minutes. Use shadcn/ui chat component. Easiest to customize visually and impress in a demo context.

Notion Sidebar

Familiar

Embed your companion's outputs into a Notion workspace. Morning briefs, job tracker, contacts — all in one place users already use.

Mobile App

Phase 2

React Native via Expo. Build after MVP. Unlocks push notifications, voice input, and home screen presence.

Conversation Design Patterns

✅ Do: Structured Outputs

When reporting email or calendar data, always use structured cards (icons, bold labels, bullet points). Never dump raw text. Structure = readable at a glance.

❌ Don't: Open-Ended Check-ins

Don't ask "How are you feeling?" — ask "On a scale of 1–5, how's your energy today?" Specific prompts get useful data and feel less awkward.

✅ Do: Confirm Before Acting

Before sending an email or creating a calendar event, always show the draft and ask "Looks good to send?" This builds trust and prevents errors.

❌ Don't: Be Preachy

If the user is stressed, don't say "You should practice self-care." Say "That sounds like a lot. Want me to clear your afternoon?" Action, not advice.

non-urgent tasks to later in the week."

Memory Injection Pattern

This is the core pattern that gives Claude "memory." Run this before every important API call.

memory_inject.js · Node.js — runs in n8n Code node 
// Fetch user context from Supabase before calling Claude
// Paste this into an n8n "Code" node before your Claude HTTP node

const SUPABASE_URL = 'https://your-project.supabase.co';
const SUPABASE_KEY = '{{ $env.SUPABASE_ANON_KEY }}';

// Fetch recent mood (last 3 days)
const moodRes = await fetch(
  `${SUPABASE_URL}/rest/v1/mood_logs?select=*&order=logged_at.desc&limit=3`,
  { headers: { 'apikey': SUPABASE_KEY, 'Authorization': `Bearer ${SUPABASE_KEY}` } }
);
const moods = await moodRes.json();

// Fetch pending follow-ups
const followupRes = await fetch(
  `${SUPABASE_URL}/rest/v1/contacts?status=eq.new&follow_up_date=lte.${new Date().toISOString()}`,
  { headers: { 'apikey': SUPABASE_KEY, 'Authorization': `Bearer ${SUPABASE_KEY}` } }
);
const followups = await followupRes.json();

// Build context string to inject into Claude prompt
const context = `
RECENT MOOD: ${moods.map(m => `Day: energy ${m.energy_score}/5`).join(', ')}
PENDING FOLLOW-UPS: ${followups.map(f => f.name).join(', ') || 'None'}
`;

return [{ json: { context } }];
🧠

What to Store vs. What to Discard

Store: mood scores, contact names, job stages, event titles, user preferences.
Discard: full email bodies (use summaries only), raw calendar descriptions, anything not needed after 90 days.

📊

Useful Supabase Queries

Average mood this week: SELECT AVG(energy_score) FROM mood_logs WHERE logged_at > now() - interval '7 days'

Jobs with no activity in 7+ days: add a WHERE clause on last_contact.
